## Overview

**Algorithms** are step-by-step procedures for solving problems, while **Big O notation** is how we measure their efficiency. Understanding both is essential for writing performant code, passing technical interviews, and making informed decisions about data processing approaches.

<Info>
**What you'll learn in this guide:**
- Big O notation and complexity analysis
- Time complexity vs space complexity
- Common complexities: O(1), O(log n), O(n), O(n log n), O(n²), O(2ⁿ)
- Sorting algorithms (bubble, merge, quick sort)
- Searching algorithms (linear, binary search)
- Common algorithm patterns (sliding window, two pointers, etc.)
- How to analyze your own code's complexity
</Info>

<Info>
Common Big O complexities from fastest to slowest: O(1) constant, O(log n) logarithmic, O(n) linear, O(n log n) linearithmic, O(n²) quadratic, O(2ⁿ) exponential.
</Info>
